==============================================

ARQUIVOS CRIADOS:
─────────────────
1. CREDIT_LOCKS_MIGRATION.sql
   - Cria tabela credit_locks (lawyer_id, case_id, locked_credits, status, expires_at, consumed_at)
   - UNIQUE constraint (lawyer_id, case_id) para prevenir duplicidade
   - Índices em: lawyer_id, case_id, expires_at, status
   - RLS habilitado

2. CREDIT_LOCK_IMPLEMENTATION.md
   - Documentação técnica concisa do sistema

ARQUIVOS ALTERADOS:
───────────────────
1. store.tsx
   
   a) Interface AppContextType
      + Adicionado: openChatWithLawyer: (caseId: string, lawyerId: string) => Promise<void>
   
   b) useEffect (login)
      + Chamada: expireLockedCredits() após fetchNotifications()
   
   c) fetchUsers()
      + Sem alteração (mantém mesmo)
   
   d) expireLockedCredits() [NOVA FUNÇÃO]
      - Busca credit_locks com status ACTIVE e expires_at < now
      - Atualiza status para EXPIRED
      - Devolve locked_credits ao saldo de cada advogado
      - Executada automaticamente em login
   
   e) acceptCase() [ALTERADA]
      - Antes: Debitava 5 Juris imediatamente e registrava interesse
      - Agora: 
        ✓ Valida saldo ≥ 1 crédito
        ✓ Verifica duplicidade (UNIQUE constraint evita race condition)
        ✓ Cria credit_lock com status ACTIVE, expires_at +7 dias
        ✓ Debita apenas 1 Juris (travado, não consumido)
        ✓ Registra interesse em case_interests
   
   f) openChatWithLawyer() [NOVA FUNÇÃO]
      - Busca credit_lock ativo (status ACTIVE)
      - Valida: credit_lock existe E advogado tem saldo ≥ 3
      - Atualiza status ACTIVE → CONSUMED + consumed_at
      - Debita +3 créditos adicionais do advogado
      - Total consumido: 1 (travado) + 3 (adicionais) = 4 Juris
      - Se falhar em qualquer etapa: retorna erro sem gerar inconsistência
   
   g) Provider value
      + Incluído: openChatWithLawyer na lista de funções do contexto

REGRAS DE NEGÓCIO IMPLEMENTADAS:
────────────────────────────────
✓ 1 crédito travado ao manifestar interesse
✓ Não gasto imediatamente, apenas reservado
✓ +3 créditos consumidos ao cliente abrir chat
✓ Total: 1 (travado) + 3 (adicionais) = 4 Juris por match
✓ Devolução automática após 7 dias se cliente não abrir chat
✓ Sem saldo suficiente (< 1): não permite manifestar interesse
✓ Sem +3 créditos ao abrir chat: bloqueia e retorna erro
✓ Duplicidade prevenida via UNIQUE constraint no DB
✓ Atomicidade garantida: todas operações validadas antes de executar
